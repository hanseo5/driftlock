# Driftlock Brainstorm

Expose meaningful alternatives without drifting into implementation.

## Workflow

1. List 2-4 viable directions with tradeoffs.
2. Separate product tradeoffs from mechanical implementation choices.
3. Recommend one default direction with rationale.
4. Route unresolved branches to driftlock-grill.

## Rules

- Keep brainstorming bounded.
- Do not treat suggestions as approvals.
- Do not create a Locked Spec.

## Output

Produce `brainstorm-notes.md`. Read `references/brainstorm-contract.md` for the exact contract.
