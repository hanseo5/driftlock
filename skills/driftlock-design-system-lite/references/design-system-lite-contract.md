# Driftlock Design System Lite Contract

## Purpose

Create enough design direction to stop implementation from producing the wrong kind of interface.

## Required Input

Use the upstream Driftlock artifacts and evidence required by this workflow step. If the required upstream artifact is missing, stop and route backward rather than guessing.

## Required Output

Produce `design-system-lite.md` with enough evidence for the next Driftlock skill to continue without re-interrogating product intent.

## Gate Rules

- Do not create a marketing landing page unless requested.
- Do not over-specify token systems for v0.
- Preserve user-approved density and first-screen expectations.

## Next Route

Route only to the next valid Driftlock skill or to `driftlock-amend-advisor` when product intent, UX lock, or spec meaning would change.
