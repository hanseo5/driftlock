# Driftlock Design System Lite

Create enough design direction to stop implementation from producing the wrong kind of interface.

## Workflow

1. Define product feel, layout rules, components, typography, color roles, and interaction rules.
2. Bias toward the target platform and product category.
3. Mark taste choices that need approval.
4. Output design-system-lite.md.

## Rules

- Do not create a marketing landing page unless requested.
- Do not over-specify token systems for v0.
- Preserve user-approved density and first-screen expectations.

## Output

Produce `design-system-lite.md`. Read `references/design-system-lite-contract.md` for the exact contract.
