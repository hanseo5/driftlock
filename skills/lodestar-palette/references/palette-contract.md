# Lodestar Palette Contract

## Purpose

Create enough design direction to stop implementation from producing the wrong kind of interface.

## Required Input

Use the upstream Lodestar artifacts and evidence required by this workflow step. If the required upstream artifact is missing, stop and route backward rather than guessing.

## Required Output

Produce `palette.md` with enough evidence for the next Lodestar skill to continue without re-interrogating product intent.

## Gate Rules

- Do not create a marketing landing page unless requested.
- Do not over-specify token systems for v0.
- Preserve user-approved density and first-screen expectations.

## Next Route

Route only to the next valid Lodestar skill or to `lodestar-course-correct` when product intent, UX lock, or spec meaning would change.
